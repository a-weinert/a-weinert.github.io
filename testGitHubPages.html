Ä Ä; Ö Ö; Ü Ü;

Here & Auml; is followed by the iso-8859-1 Ä; and so on.
Every (Apache 2.4) web server delivers this page unspoiled but Github Pages won't.
With Github Pages all iso-8859-1 charactes above 127 get displayed as question mark in black diamond (�).

Background

Our very first (quasi hello world) experience with github pages was disapointing.
In spite of
  <?xml version="1.0" encoding="iso-8859-1"?>
and
  <meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1" />
as well as even (irrelevant for caracter set settings, of course)
  <meta http-equiv="content-language" content="de" />
all German Umlauts are spoiled.
